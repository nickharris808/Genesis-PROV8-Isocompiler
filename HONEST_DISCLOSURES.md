# Honest Disclosures

This document provides a complete and candid accounting of what IsoCompiler can and cannot do, what has been validated and what has not, and where the boundaries of our evidence lie.

---

## Simulation, Not Fabrication

**No isolation structure synthesized by IsoCompiler has been fabricated or measured.**

All performance claims (55-63 dB isolation improvement, 10 frequency bands, 5 synthesis families) are based on electromagnetic simulation, not physical measurement. The platform generates fabrication-ready packages (Gerber RS-274X, Excellon drill files, BOM), but no test structures have been ordered, fabricated, or measured with a VNA or any other instrument.

---

## 2D FDTD, Not 3D Full-Wave

The default solver is a 2D Transverse Magnetic (TMz) Finite-Difference Time-Domain (FDTD) implementation. This is a legitimate and widely used electromagnetic solver, but it has inherent limitations:

- **No out-of-plane coupling.** 2D TMz simulates an infinitely long structure in the z-direction. Real chiplet packages have finite vertical extent, and electromagnetic energy can couple over, under, and around isolation structures through out-of-plane paths.
- **No via inductance in 3D.** Via fences and mushroom EBG vias have 3D inductance that is not captured in 2D simulation. This affects bandgap predictions and isolation bandwidth.
- **No edge radiation.** Real isolation structures have finite extent in all three dimensions. Radiation from structure edges reduces effective isolation.

**Quantified impact:** The 2D solver predicts 55-63 dB isolation improvement. Analytical 3D models (parallel-plate waveguide with mode summation) predict 38.6-39.1 dB improvement. The 16-24 dB gap is attributed to out-of-plane leakage paths absent in the 2D model.

A 3D solver backend (MIT Meep) is available and has been tested with analytical validation, but has not been extensively used for isolation synthesis optimization runs.

---

## Analytical Validation, Not Measurement Validation

The FDTD solver is validated against four analytical solutions:

1. **Free-space propagation:** S21 near 0 dB (wave propagates without obstruction).
2. **PEC wall reflection:** S21 < -30 dB (perfect conductor blocks propagation).
3. **Dielectric slab (Transfer Matrix Method):** Frequency-dependent transmission matches TMM within +/- 5 dB.
4. **Substrate mode coupling:** Guided-mode estimate within +/- 15 dB.

These are standard physics benchmarks. They confirm that the solver correctly implements Maxwell's equations in discrete form. They do not confirm that the solver accurately predicts the behavior of specific isolation structures in specific packages, because that would require measurement validation against fabricated structures.

## Adjoint Gradients: Adjoint Simulation, Not Automatic Differentiation

The adjoint gradient of S21 with respect to the material distribution is computed by running a separate adjoint FDTD simulation, not by using automatic differentiation (AD) through the FDTD solver.

This is the standard approach in the computational electromagnetics and topology optimization literature (Molesky et al., Nature Photonics 2018; Hughes et al., ACS Photonics 2018). It is mathematically equivalent to reverse-mode AD, but the implementation consists of:

1. A forward FDTD simulation (standard).
2. An adjoint FDTD simulation (reverse source).
3. A field overlap integral to compute the gradient.

This is not the same as tracing every FDTD timestep through an AD framework (such as JAX's autodiff). The AD approach would provide exact gradients including all numerical effects; the adjoint simulation approach provides mathematically correct gradients of the continuous problem discretized on the same grid.

In practice, the difference between adjoint-simulation gradients and AD gradients is negligible for the grid resolutions used in IsoCompiler. But we disclose the implementation approach for completeness.

## GDSII Tested with KLayout, Not Foundry DRC

All GDSII output has been verified with KLayout DRC runsets generated by IsoCompiler. The DRC rules check:

- Minimum line width
- Minimum space between features
- Minimum via diameter
- Keepout zone compliance

These are basic geometric DRC checks. They have **not** been tested against production foundry DRC decks from TSMC, Intel, Samsung, GlobalFoundries, or any OSAT. Production foundry DRC includes hundreds of additional rules (antenna rules, density rules, metal fill, electromigration, stress, etc.) that are not checked by IsoCompiler's DRC.

A real tapeout would require importing IsoCompiler's GDSII output into a commercial DRC tool (Synopsys ICV, Cadence PVS, or Siemens Calibre) and running it against the target foundry's full DRC deck.

## S21 Numbers: Context and Caveats

The 55-63 dB isolation improvement figures represent the difference in S21 (transmission coefficient) between:

- **Baseline:** Unmitigated substrate coupling (no isolation structure).
- **Optimized:** Substrate with IsoCompiler-synthesized isolation structure.

Caveats:

1. These are 2D FDTD results (see above re: 3D gap).
2. S21 extraction uses Hanning-windowed FFT with spectral quality validation. Each measurement is categorized as GENUINE, MARGINAL, or ARTIFACT based on source energy at the extraction frequency. Previous versions of the platform had a fatal frequency normalization bug that produced noise-floor artifacts masquerading as S21 measurements; this has been corrected.
3. The improvement range (55-63 dB) varies by frequency band and structure type. Not all structure types achieve maximum isolation at all frequencies.

## Material Library: Validated Models, Not Measured Data

The dispersive material library includes 20+ materials with frequency-dependent permittivity and loss:

- **Metals:** Drude model (copper, aluminum, gold, silver, tin, nickel).
- **PCB substrates:** Debye model (FR4, Rogers 4350B, Megtron 6/7, polyimide).
- **Semiconductors:** Simple and Debye models (silicon, SiO2, Si3N4).

These models use published material parameters from datasheets and textbooks. They have not been fitted to measured data from specific production lots. Real material properties vary with manufacturer, process conditions, temperature, and frequency. The models are representative, not exact.

---

## VNA Measurement Prediction: A Model, Not Data

The VNA measurement simulation models a Keysight PNA-X N5247B measuring fabricated test structures. It includes cable loss, SOLT calibration residuals, connector repeatability, noise floor, and temperature drift. This is a physics-based prediction of what measurement would show, not actual measured data. It has not been validated against real VNA measurements.

## Single-Developer Codebase

IsoCompiler was developed by a single engineer. While the 132-test suite provides reasonable coverage, the platform has not undergone:

- Multi-person code review
- Production deployment at scale
- Stress testing with real customer workloads
- Security audit
